# Efficio: Claude Code 세션의 크기보정 낭비 신호에 대한 0-토큰 자기평가 프레임워크

> 짧은 논문(초록급) 정리 — 설계 문서 `docs/waste-aware-eval-design.md`(v0.2.1)와
> 구현(`efficio/`)의 실제 산식·검증 수치를 근거로 정리. 대상: 외부 공유용 요약.

---

## 초록 (Abstract)

대규모언어모델(LLM) 코딩 에이전트의 "효율"을 측정하려는 기존 시도는 산출물(PR 수·LOC)
또는 투입(토큰·시간)을 측정하나, 두 측정 모두 **세션 규모와 교란**되어 "헛수고"를
분리하지 못한다. 본 연구는 같은 결과를 내는 데 쓰지 않아도 됐을 토큰·반복·재작업, 즉
**회피 가능한 낭비(avoidable waste)**의 *크기보정 잔차*를, 평가 시스템 자체가 LLM을 전혀
호출하지 않고(0 토큰) Claude Code 세션 transcript(JSONL)에서 결정론적으로 산출하는
방법을 제시한다.

핵심 설계는 세 가지다. (1) 원시 낭비 신호 *W*를 세션 크기 *s = log(total_tokens)*에
**Theil–Sen 로버스트 회귀**시킨 잔차 *r = W − (â + b̂·s)*로 정규화하여 "규모"가 아닌
"낭비"를 분리한다. (2) 회귀계수와 기준 잔차 분포를 **버전 고정 기준 모델**로 영속화하여,
코퍼스가 늘어도 동일 세션 점수가 드리프트하지 않는 재현성을 보장한다. (3) 단일 종합점수
대신 **다축 프로파일**로 제시한다.

균질 개발세션 *n=184*의 Pilot-0 실측에서 원시 신호는 크기에 지배되나(raw PCA의
PC1 = 78.8%), 잔차화 후 크기상관은 *ρ = 0.10–0.13*으로 떨어지고 신호 분산의 66–79%가
보존되었다. 기준타당도 검증(MTMM-lite, 3방법)에서 **단일 주축은 존재하지 않았다**:
주관 라벨은 컨텍스트 재무효화(W2)와 수렴(*ρ = +0.57*)한 반면, 두 독립 객관 신호(세션간
rework·세션내 Bash 반복)는 서로 수렴(*ρ = +0.43*)하면서 주관 라벨과는 음의 방향
(*ρ = −0.52, −0.36*)으로 갈렸다. 이는 **체감 낭비와 행동 낭비가 실재하는 별개 2차원**임을
시사한다(Campbell & Fiske 방법분산 분리). 따라서 어느 축이 "진짜 낭비"인지는 가치판단으로
남기고, 두 축군을 분리 표기한다.

한계로 (a) attribution 오귀속률 55.7%로 PR 단위 측정(W1·W4·생존율)은 보류되어
작업단위가 단일 세션으로 한정되며, (b) ρ≥0.5 임계의 견고한 통과는 미확정(CI 하한
0.18–0.21), (c) 방향 타당성은 측정하지 않아 잘못된 방향으로 정확히 수행한 세션은
거짓양성이 될 수 있다.

## 1. 문제 정의와 측정 대상

> **정의 (회피 가능한 낭비).** 그 시점에 이미 확보 가능했던 정보를 고려할 때, 일어나지
> 않아도 됐을 토큰·반복·재작업.

무료 결정론 층은 이를 직접 재지 못한다 — "회피 가능" 여부는 반사실(counterfactual)
기준선과 LLM 판별을 요구하기 때문이다. 따라서 무료 층은 **"총 폐기·재탐색·반복의
크기보정 잔차"**만 측정한다고 정직하게 표기한다. 작업 단위(WU)는 PR↔세션 연결의 결정론적
불가능성(Pilot-0 오귀속률 55.7%) 때문에 **단일 세션**으로 고정한다.

명시적 비목표: 인사고과·승진·보상 입력값 아님, 개인 간 순위 아님, 감시 도구 아님,
방향·전략의 타당성 평가 아님(실행 효율만 측정).

## 2. 계산 모델

### 2.1 원시 신호 추출 (0 토큰)

검증 범위는 균질 개발세션(`turns ≥ 3 ∧ assistant ≥ 4`)으로 한정한다. 1턴 리뷰 등 신호가
빈 세션은 제외. transcript JSONL에서 다음 네 축의 원시 신호를 추출한다:

```
W2_raw   = Σ cache_creation_tokens − (첫 어시스턴트 메시지의 cache_creation)
           (초기 워밍업=회피불가 제외. 세션 중반 이후 거대·변동 컨텍스트
            재주입으로 캐시가 반복 무효화된 토큰만 포착)
W3_raw   = |read_paths| − |set(read_paths)|     (동일 파일 반복 read)
WC_raw   = |edit_paths| − |set(edit_paths)|     (동일 파일 반복 edit)
Bash_raw = |bash_cmds|  − |set(bash_cmds)|      (동일 Bash 명령 반복)
```

구현: `efficio/signals.py`. 동일 message.id 반복 블록은 last-wins로 dedup, 비공식
JSONL 포맷은 방어적 파싱(라인 단위 try/except).

### 2.2 크기보정 잔차화 (Theil–Sen)

원시 신호는 크기에 종속되므로(Pilot-0: raw ρ = 0.74–0.94), 크기 인자에 회귀시킨 잔차만
사용한다. 분포가 비정규·이상치를 포함하므로 OLS 대신 Theil–Sen을 쓴다:

```
b̂ = median_{i<j} (y_j − y_i) / (s_j − s_i)      # 모든 쌍 기울기의 median
â = median_i (y_i − b̂·s_i)
s = log(total_tokens + 1)                        # 13장 PCA의 PC1 대용
```

세션의 **반사실 기준선**(같은 크기의 예상 신호)과 **낭비 잔차**는:

```
baseline = â + b̂·s
r        = W_raw − baseline
```

부호 규약: r > 0 = 같은 크기 대비 신호 초과(낭비 의심), r < 0 = 미만.
구현: `efficio/residual.py`.

### 2.3 자기 대비 백분위와 영과잉 처리

자기 코퍼스의 기준 잔차 분포 R_ref 대비 백분위로 채점한다:

```
pct(r) = |{ x ∈ R_ref : x < r }| / |R_ref| × 100
```

W3·WC는 영과잉(median=0)이므로 `is_zero` 플래그로 "0=발생 안 함 vs 0=데이터 없음"을
구분(허들 2부 모델: ① P(신호>0) ② 신호>0일 때 크기보정 크기).

### 2.4 재현성 — 기준 모델 고정

매 조회마다 전체 코퍼스로 재적합하면 세션 증가 시 동일 세션 점수가 드리프트한다(재현성
결함). 따라서 (â, b̂, R_ref)를 **버전 고정 기준 모델**로 영속화(`fit_reference`)하고,
채점은 적용만(`apply_reference`) 한다. 재적합은 명시적 행위로 옛 버전을 보존하여 옛
점수를 재현한다. 구현: `efficio/reference.py`, 테스트 `efficio/tests/test_reference.py`.

## 3. 측정 축과 검증 상태

| 축군 | 축 | 의미 | 검증 상태 |
|---|---|---|---|
| **체감** | W2 컨텍스트 재무효화 | 워밍업 제외 cache_creation 잔차 | 주관 H1 상관(ρ≈0.57)·객관과 무관 |
| **행동** | WC 편집 반복 | 동일 파일 반복 edit | 객관 rework와 약상관(ρ≈0.19) |
| **행동** | Bash 시행착오 | 동일 Bash 명령 반복 | 객관 rework와 수렴(ρ≈0.43) |
| 행동 | W3 재탐색 | 동일 파일 반복 read | 미검증 |

기본 표시 축은 W2(검증 우위가 아니라 신호 밀도가 높아 기본값). 단일 '주축' 없음.

## 4. 검증 결과 (요약)

**Pilot-0 (n=184 균질 개발세션):**

| 검증 | 방법 | 결과 | 잠정 임계 | 판정 |
|---|---|---|---|---|
| attribution 오귀속률 | cwd+시간창 휴리스틱 | 55.7% | ≤10% | FAIL → WU=단일세션 |
| 신호 밀도(비-0 비율) | W2/W3/WC | 100% / 52.2% / 62.5% | ≥30% | PASS(3축) |
| 구성타당도(raw) | PCA PC1 | 78.8% | ≤50% | FAIL → 잔차화 필수 |
| 잔차 크기상관 | W2/W3/WC | 0.10/0.13/0.12 | <0.3 | PASS |
| 신호 보존량 | 잔차/원시 분산 | 0.72/0.79/0.66 | ≥0.5 | PASS |

**MTMM-lite 기준타당도 (n=24–30, 부트스트랩 B=3000):**

| 교차 | ρ | 95% CI |
|---|---|---|
| 주관 라벨 ↔ W2 | +0.57 | [+0.18, +0.82] |
| M2 rework ↔ M3 Bash반복 (객관–객관) | +0.43 | [+0.09, +0.71] |
| 주관 라벨 ↔ M2 rework | −0.52 | [−0.86, −0.11] |
| 주관 라벨 ↔ M3 Bash반복 | −0.36 | [−0.66, −0.01] |

→ 두 객관 신호가 서로 수렴하며 주관과 반대로 갈림 = **체감축(W2) / 행동축(WC·Bash)의
실재하는 2차원 구조**. METR(2025)의 체감/실제 39%p 괴리와 정합.

## 5. 산출 인터페이스

```bash
efficio collect              # 세션 스캔 → ~/.efficio/efficio.db (모델 없으면 초기 적합)
efficio fit                  # 기준 모델 재적합(드리프트 통제)
efficio timeline --last 20   # W2 잔차 백분위 시계열 (백분위↑=낭비↑)
efficio profile <id>         # 세션별 다축 효율 프로파일 (실제 vs 기준선)
```

프로파일은 단일 점수를 내지 않고, 각 축을 `value ± band`,
`data_quality{otel, hook, git}`, `is_zero_real`, 검증상태(주관검증/객관약검증/미검증)와
함께 병치한다. 축별 막대를 상호 비교 가능하게 병치하지 않는다(이질 단위·분포라 막대 길이
비교가 아티팩트).

## 6. 한계와 비목표

- **방향 타당성 미측정**: 잘못된 방향으로 정확히 수행한 세션은 거짓양성(범위 밖 명시).
- **임계 미확정**: ρ≥0.5를 W2(0.57)가 점추정으로 넘으나 CI 하한 0.18로 견고히 넘는다
  단정 불가. W3·WC는 개별 기준타당도 약함.
- **구성타당도 ≠ 기준타당도**: 잔차가 "크기가 아님"은 보였으나 "실제 헛수고임"(H1)은
  부분 검증.
- **단일 사용자·자기 프로젝트 파일럿**(l2u-2nd 58/184 편중) → 타 사용자 일반화는 별도
  표본 필요.
- **비목표**: 인사고과·개인 순위·감시 도구 아님. 셀프 전용(M0–M2), 팀 집계는 k-익명성
  기술강제 이후(M3).
