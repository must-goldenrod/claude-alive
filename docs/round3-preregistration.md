# H1 라운드3 — 사전등록 (Pre-registration)

> 데이터(라벨) 수집 *전에* 가설·임계·표본·분석을 고정한다. 사후 임계 설정의 순환논증을
> 막기 위함(Lakens et al. 2018; Lakens 2022). 라운드1·2의 약점(저검정력·라벨압축·단일평정자·
> 사후임계)을 방법론 리뷰(`docs/methodology-review.md`) 기준으로 교정한 설계.
> 상태: **등록(미실행)** — 라벨링은 사용자 입력 대기.

## 1. 검증 가설

- **H1(주판정):** 잔차 낭비 신호가 사람의 *기계적 헛수고* 평정과 양의 상관.
- **H1c(다방법 수렴):** 두 객관 방법(rework·Bash)이 서로 수렴하고, 주관 라벨과의 관계가 라운드2(음의 갈림)에서 재현되는가.

## 2. SESOI·판정 임계 (사전 고정)

- **SESOI = Spearman ρ ≥ 0.3** ("moderate", Ferguson & Heene 2021 r=0.1 무관 하한보다 위). ρ≥0.5는 폐기(관측치 근처·표본 폭발 n≈400).
- **합격:** 부트스트랩(BCa, B=5000) **95% CI 하한 > 0.3** (우월성). 다축은 Bonferroni.
- **반증:** CI가 0을 포함하거나 하한 ≤ 0 → 해당 축 기각.

## 3. 표본 (AIPE 기반)

- **목표 CI 폭 ≤ 0.4** → AIPE n ≈ **62**(Bonett-Wright 보정 포함 ~+10% → **≈70**). 여유 목표 n=80.
- 동질 개발세션(turns≥3, assist≥4), 잔차 신호 범위를 고르게 가르도록 선정(기존 `h1b_labeling.py` 확장).

## 4. 라벨링 프로토콜 (라운드2 약점 교정)

- **전체 1~5 척도 강제 사용**: 1=거의 한 번에 매끄러움 … 5=같은 걸 여러 번 다시/헤맴. **각 점수 최소 비율 권고**(예: 1·5 각 ≥10%)로 압축 방지(라운드2는 1~3만 써 정보 47%·동점 46%).
- **기준 정렬(라운드2 유지):** 결과·방향 무시, *기계적 헛수고만*. 방향 타당성은 측정 안 함.
- **≥2 평정자**, 서로 독립. 사적 세션이라 본인+1인이 어려우면, **객관 RW를 제2 평정자 대용**으로 명시(삼각검증).
- **블라인드:** 신호 점수를 숨긴 시트로 라벨(공통방법편향 통제, Podsakoff 2003).

## 5. 분석 계획 (사전 고정)

1. **상관:** Spearman, **BCa 부트스트랩 CI**(동점·소표본에서 Fisher-z 해석적 CI 부정확 — Ornstein & Lyhagen 2016).
2. **부분상관:** 신호·라벨 **양변수**를 log(tokens)에 잔차화(partial). **over-control 민감도** 동반 보고(raw vs 잔차; 라운드2에선 잔차화가 신호 강화 → confounder 확인).
3. **평정자 신뢰도:** ICC(2,1) + 2차 가중 κ(`multirater_efa_tools.py`). **약화보정** ρ_true=ρ_obs/√reliability (소표본 과교정 주의 → CI 병행).
4. **다방법(MTMM):** M1 주관·M2 rework·M3 Bash 3방법 수렴 행렬(`mtmm_convergent.py`). 두 객관 수렴 + 주관 갈림이 재현되는지.
5. **구성타당도:** EFA(공통요인) + 잔차 크기상관 |ρ|<0.3.

## 6. 사전 예측 (라운드2 기반)

- W2(체감축): 주관 라벨과 ρ≈0.4~0.5, 객관 RW와 ~0. → H1 주판정은 **주관 기준에서** 통과 예상, 객관 기준에선 미통과 예상.
- WC·Bash(행동축): 객관 RW와 ρ≈0.2~0.4, 주관과 약/음.
- **가장 그럴듯한 결론:** "약~중간 강도의 진짜 신호, 단 체감/행동이 갈림" — 강한 단일 측정도구는 아님(검정력은 *확정*을 줄 뿐 효과를 키우지 못함).

## 7. 결정 규칙

- W2가 주관 SESOI 통과 + 행동축이 객관 SESOI 통과 → 도구의 2축군(체감/행동) 유지·"검증" 격상.
- 둘 다 미통과 → 신호 재설계 또는 제품 전제 재검토.
- 부분 통과(현재 예상) → 통과 축만 "검증", 나머지 "실험" 유지 + 한계 명시.

## 8. 한계 (사전 인지)

단일 사용자·자기 프로젝트(외적 타당도 미확보 — 타 사용자 표본 별도 필요). RW·Bash는 노이즈 프록시(리팩터링 교란 — 8h 필터 변형 병행). 기준선은 "크기 대비 예상"이지 이상적 최소 궤적 아님(12장 Q1).
